Features
Built for privacy by design
🛡️
Zero-Knowledge Server

All cryptographic operations happen in your browser. The server stores only ciphertext it mathematically cannot decrypt. By design, not policy.

⚛️
NIST 2024 Standards

ML-KEM (FIPS-203) for encryption and ML-DSA (FIPS-204) for signatures — the same algorithms adopted by Google, Cloudflare and the US federal government.

🔗
Universal Shortlinks

Every message gets a shortlink. Paste it in Gmail, WhatsApp, Slack, SMS. Recipients need no account to decrypt — just their private key.

🔥
Burn After Read

Messages are permanently deleted the moment the recipient opens them. Nothing lingers on our servers beyond what you allow.

📦
No File Size Limits

Files are split into 3MB encrypted chunks and stored in MongoDB GridFS. Upload gigabytes — no single request ever approaches Cloudflare limits.

🔌
Embeddable Widget

Add one line of code to any website. Opens an inline modal with the full encryption flow — no redirect, no new tab. Perfect for onionmail.org and similar privacy services.

📜
Quantum-Safe Notary

Document hash + ML-DSA signature + server timestamp = a Notary Receipt. The document never leaves your browser. Anyone can verify authenticity with the receipt ID.

👤
Public Profiles

Register a username. Share pqcserver.com/u/you — anyone can send you encrypted messages without knowing your key.

🌐
REST API

Integrate PQCServer into your own applications. Upload files, send messages and verify notary receipts programmatically without building the cryptography yourself.

The threat is happening now — not in the future

This attack — called "Harvest Now, Decrypt Later" — means data encrypted with RSA or ECC right now is already at risk. PQCServer uses ML-KEM and ML-DSA — NIST-standardized algorithms designed to resist both classical and quantum computers.
